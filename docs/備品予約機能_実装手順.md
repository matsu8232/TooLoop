# 備品の1日単位予約機能 - 実装手順

参考: [Railsのsimple_calendarをカスタマイズして時間帯で予約できる予約機能を作ってみた！](https://qiita.com/sssssatou/items/2e6606e3ddf9b246a0fb)

本プロジェクトでは**時間単位ではなく1日単位**で備品を予約するため、記事の「時間帯」部分は省き、**日付のみ**で予約可能かどうかを判定します。

---

## 前提・現状

- **Reservation** モデル: `user_id`, `item_id`, `start_date`, `end_date` あり
- **1日予約**: `start_date` と `end_date` を同じ日付で保存する運用にする
- **simple_calendar**: Gemfile に既に導入済み。`items/show` で `month_calendar` 使用中
- **ルート**: `reservations` のルートは未定義

---

## 手順1: ルートの追加

備品（item）に紐づく予約として、**ネストしたリソース**にします。

```ruby
# config/routes.rb
resources :items do
  resources :reservations, only: [:index, :new, :create, :show, :destroy]
end
```

- 予約一覧・カレンダー: `GET /items/:item_id/reservations`
- 新規予約画面: `GET /items/:item_id/reservations/new?day=2026-03-20`
- 予約作成: `POST /items/:item_id/reservations`
- 予約詳細: `GET /items/:item_id/reservations/:id`
- 予約キャンセル: `DELETE /items/:item_id/reservations/:id`

---

## 手順2: Reservation モデルの整備

### 2-1. バリデーション

- `start_date`, `end_date` の presence
- **1日単位**のため、`start_date == end_date` とするか、または「予約日」を `start_date` のみで扱い `end_date` は `start_date` と同じ日で保存
- **過去日は予約不可**
- **同一 item ・同一日付の重複予約を禁止**（1備品1日1予約）

### 2-2. クラスメソッド（予約済み日一覧）

カレンダー表示用に「その備品の、指定期間内の予約済み日」を返すメソッドを用意します。記事の `reservations_after_three_month` と同様に、**日付の配列**または **日付＋必要情報の配列** を返すと便利です。

例: 今日から3ヶ月先までで、ある item の予約済み日を `YYYY-MM-DD` の配列で返す。

### 2-3. 日付チェック用メソッド（任意）

新規予約画面に遷移する前に「過去日でないか」「既に予約が入っていないか」をチェックするメソッドがあると、記事のようにリダイレクト＋flash でエラーを出せます。

---

## 手順3: ReservationsController の実装

| アクション | 役割 |
|-----------|------|
| **index** | 対象の `@item` を取得し、その item の予約を simple_calendar 用に取得。`month_calendar` で表示。 |
| **new** | `params[:day]` で日付を受け取り、過去日・重複のチェック。問題なければ `@reservation`（item, day をセット）を用意。 |
| **create** | `reservation_params` で `user_id`, `item_id`, `start_date`, `end_date`（同じ日付）を保存。成功時は show または index へリダイレクト。 |
| **show** | 予約詳細表示。 |
| **destroy** | キャンセル（削除）。 |

`new` では、記事と同様に「過去日・当日制限・3ヶ月以降」などをチェックし、NG なら `redirect_to item_reservations_path(@item), flash: { alert: message }` のようにするとよいです。

---

## 手順4: Helper の実装

### 4-1. 予約済みかどうか判定するメソッド

記事の `check_reservation(reservations, day, time)` を、**1日単位**用に簡略化します。

- 引数: 「その item の予約済み日一覧（または予約オブジェクトの配列）」と「対象の日付」
- 返り値: その日が予約済みなら `true`、空いていれば `false`

これでカレンダー上で「○（リンク）／×（予約済み）」を切り替えられます。

---

## 手順5: ビューの実装

### 5-1. simple_calendar のビューを生成（任意）

カスタマイズする場合は以下でビューを生成します。

```bash
rails g simple_calendar:views
```

生成される `app/views/simple_calendar/_month_calendar.html.erb` を編集して、各日に「予約済みなら×、空いていれば new へのリンク」を出します。1日単位なので、記事のような「時間帯の行」は不要で、**日付のセルの中だけ**判定すれば十分です。

### 5-2. reservations#index

- 対象の `@item` の名前などを表示
- `month_calendar events: @reservations` でカレンダーを表示
- 各日付セル内で、helper の「予約済みか」メソッドで分岐
  - 予約済み: 「×」や「予約済」などのテキスト
  - 空き: `link_to '○', new_item_reservation_path(@item, day: date)` のように日付を渡す

`@reservations` は、その item の「表示期間内の予約」を、simple_calendar が期待する形（例: `start_time` を持つレコード）で渡す必要があります。1日単位の場合は、各予約の `start_date` を `start_time` に合わせるか、またはカスタムビューで `start_date` を参照するようにします。

### 5-3. reservations#new

- `@day`, `@item` を表示
- `form_with model: @reservation, url: item_reservations_path(@item)` でフォーム
- `hidden_field` で `start_date`, `end_date`（同じ日）、`user_id`, `item_id` を送信
- 「予約する」ボタン、戻るリンク（`item_reservations_path(@item)`）

### 5-4. reservations#show

- 予約日、備品名、予約者など表示
- 「予約一覧に戻る」→ `item_reservations_path(@item)`
- 自分の予約なら「キャンセル」→ `destroy`

---

## 手順6: simple_calendar と「日付」の扱い

simple_calendar の `month_calendar` は、デフォルトで **イベントに `start_time`** を期待します。1日単位で「日付」だけ使う場合の選択肢は次の2つです。

1. **Reservation に `start_time` を用意する**  
   - マイグレーションで `start_time`（datetime）を追加し、予約作成時に「その日の 0:00」などでセットする。  
   - こうすると `month_calendar events: @reservations` がそのまま使え、`start_time` の日付でカレンダー上に表示される。

2. **カスタムビューで日付のみ使う**  
   - `rails g simple_calendar:views` で生成した `_month_calendar.html.erb` を編集し、`date_range` と「その item の予約済み日一覧」を比較して、各セルに ○/× を表示する。  
   - この場合は「イベント」として @reservations を渡さず、日付ループと予約済み日リストだけで描画する。

既存の `start_date` / `end_date` だけでも実装可能なので、**手順2の「予約済み日一覧」メソッド**と**手順4の「その日が予約済みか」判定**を用意し、カレンダーは「日付ループ ＋ 予約済み日リスト」で描く形が分かりやすいです。

---

## 手順7: 備品詳細画面から予約へ誘導

`app/views/items/show.html.erb` のカレンダー横（または下）に、

- 「この備品を予約する」→ `item_reservations_path(@item)`（予約一覧・カレンダー）

のリンクを追加します。ログイン済みかつ「自分以外の備品」のときだけ表示するなど、条件を付けてもよいです。

---

## 手順8: その他

- **認証**: 予約の new / create / destroy は `before_action :authenticate_user!` でログイン必須にする。
- **権限**: 他人の予約の show / destroy は見せない・削除できないようにする。
- **flash**: 作成成功・キャンセル成功・バリデーションエラーを flash で表示する。

---

## 実装の流れまとめ

| 順番 | 内容 |
|-----|------|
| 1 | routes に `resources :items do resources :reservations ... end` を追加 |
| 2 | Reservation にバリデーション・予約済み日一覧メソッド・日付チェックメソッドを追加 |
| 3 | ReservationsController に index, new, create, show, destroy を実装 |
| 4 | ReservationsHelper に「その日が予約済みか」を判定するメソッドを追加 |
| 5 | （必要なら）`rails g simple_calendar:views` でカレンダービューをカスタム |
| 6 | reservations の index / new / show ビューを作成し、カレンダーで ○/× と new へのリンクを実装 |
| 7 | items/show から「この備品を予約する」リンクを追加 |

この順で進めれば、記事の考え方を「1日単位」に簡略化した備品予約機能を実装できます。
